- [ ] Make `ru` the built-in default locale and fallback message tree for Tinyfy. Make complete built-in `en` messages selectable with `locale="en"` without requiring the host to pass an English override dictionary. For unknown locales and missing host overrides, fall back per key to Russian.
- [ ] Retain `EditorMessageTree` and `EditorMessageKey` as the type-derived canonical contract. English and Russian must have identical string-leaf keys; host catalogs remain recursively partial overrides and overlay the selected built-in locale without mutating it.
- [ ] Keep product wording stable except where direct Russian translation, grammar, keyboard-search terms, or plural wording requires language-specific text. Reuse the same message key for the same semantic control; keep distinct keys when the audit identifies different presentation surfaces.
