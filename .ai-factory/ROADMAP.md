# Project Roadmap

> План развития редактора (Vue 3 + TypeScript + Tiptap v3) как **встраиваемой
> библиотеки для сервиса Tinyfy**, развиваемой в отдельном репозитории.
> Пункты расположены в логическом порядке выполнения: инфраструктура качества →
> типизация → тестовый фундамент → границы библиотеки → пакетизация и публикация →
> интеграция в Tinyfy → Vue-практики → производительность → локализация →
> качество кода → технический долг → развитие (коллаборация, AI).
>
> Принятые продуктовые решения (2026-07-05):
> - **Назначение**: редактор — компонент кабинета Tinyfy (Vue 3). Читатели
>   опубликованных страниц получают **статический HTML без JS редактора**
>   (принцип Tinyfy «скорость — это статус», TTFB < 50 ms).
> - **Форма поставки**: монорепозиторий из пакетов
>   `@i-prikot/editor-schema` (изоморфный набор расширений, без Vue),
>   `@i-prikot/editor` (Vue-компонент + стили),
>   `@i-prikot/editor-renderer` (серверный рендер Tiptap JSON → HTML)
>   и `apps/playground` (текущее демо-приложение для разработки).
> - **Источник истины документа**: Tiptap JSON + поле `schemaVersion`;
>   хранение — бэкенд Tinyfy (Node.js + MySQL). HTML публичной страницы —
>   производный артефакт, генерируемый `@i-prikot/editor-renderer`.
> - **AI**: платное расширение Tiptap Pro AI не используется; в перспективе —
>   собственное AI-расширение (Этап 11).
> - **Коллаборация**: внедряется позднее, после появления собственной
>   бэкенд-части (Этап 11).
> - **Вставка таблиц**: из Google Sheets — работает; Excel — не проверено
>   (нет установленного Excel).

## Milestones

### Этап 8. Локализация (i18n)

> Аудитория Tinyfy — русскоязычные эксперты: `ru` — приоритетный язык интерфейса.

- [ ] Проанализировать все пользовательские строки в проекте (тексты кнопок, tooltips, aria-label, placeholder `'Start writing...'`, пункты slash/drag-меню, названия цветов)
- [ ] Выбрать механизм локализации библиотеки: словарь сообщений через prop/options (без жёсткой зависимости от Vue I18n хоста) либо Vue I18n как peer — зафиксировать решение
- [ ] Создать базовую структуру файлов переводов с разбивкой по неймспейсам (toolbar, menus, table, image, errors)
- [ ] Добавить поддержку русского (`ru`, приоритет) и английского (`en`) языков
- [ ] Вынести все строки интерфейса в систему локализации (компоненты `notion/`, `ui/`, `table/`, primitives)
- [ ] Заменить «захардкоженные» строки в данных меню (`slash-menu-items.ts`, пункты DragContextMenu/TableHandleMenu, названия цветов в палитрах) на ключи локализации
- [ ] Локализовать динамические строки: подписи аватаров, счётчики соавторов, сообщения об ошибках загрузки изображений
- [ ] Язык задаёт хост (prop `locale`); в playground — переключатель с сохранением в `localStorage`
- [ ] Обеспечить лёгкое добавление новых языков: типизация ключей переводов, документация «как добавить язык», проверка полноты словарей в CI
- [ ] Добавить тесты для проверки локализации ключевых компонентов (рендер в `ru`/`en`, отсутствие «сырых» ключей в DOM)
